# 0.6 – UI Style Setup (Tailwind + Outfit)

Goal: wire up your design tokens (type + color) in the Next.js app so you can start building screens with consistent, reusable utilities and classes.

## 2) Add Outfit as the app font

Use Next’s built-in font loader:

```ts
// app/layout.tsx
import "./globals.css";
import { Outfit } from "next/font/google";

const outfit = Outfit({
  subsets: ["latin"],
  variable: "--font-outfit",
  display: "swap",
  // Include full range so “Black” weight is available for headings
  weight: ["300","400","500","600","700","800","900"],
});

export default function RootLayout({ children }: { children: React.ReactNode }) {
  return (
    <html lang="en" className={outfit.variable}>
      <body>{children}</body>
    </html>
  );
}
```

## 3) Global base styles and CSS variables

```css
/* app/globals.css */
@tailwind base;
@tailwind components;
@tailwind utilities;

/* Base defaults */
@layer base {
  :root {
    /* semantic aliases (use or extend as needed) */
    --bg:        theme(colors.white);
    --fg:        theme(colors.black);
    --brand:     theme(colors.ribbon.500);
    --accent:    theme(colors.flamingo.400);
    --muted:     theme(colors.vulcan.500);
  }

  html, body {
    @apply bg-[var(--bg)] text-[var(--fg)];
    @apply font-sans antialiased;
  }

  /* Default typography rhythm */
  p { @apply leading-body; }
}
```

## 4) Reusable typography utilities

Create a small **`components/typography.css`** (or co-locate as you prefer) to map your tokens to easy classes:

```css
@tailwind components;

@layer components {
  /* Headings (Outfit Black, -2% tracking, 104% leading) */
  .hd-1 { @apply text-h1 font-black tracking-hd; }
  .hd-2 { @apply text-h2 font-black tracking-hd; }
  .hd-3 { @apply text-h3 font-black tracking-hd; }
  .hd-4 { @apply text-h4 font-black tracking-hd; }
  .hd-5 { @apply text-h5 font-black tracking-hd; }
  .hd-6 { @apply text-h6 font-black tracking-hd; }

  /* Body sizes (140% leading) */
  .text-2xlb { @apply text-2xlb; }
  .text-xlb  { @apply text-xlb;  }
  .text-lgb  { @apply text-lgb;  }
  .text-baseb{ @apply text-baseb;}
  .text-smb  { @apply text-smb;  }
  .text-xsb  { @apply text-xsb;  }
}
```

Import it once (e.g., in `app/layout.tsx` with `import "@/components/typography.css";` or by referencing from `globals.css`).

## 6) Color usage guide (quick)

* **Primary brand**: `ribbon.500` (`text-ribbon-500`, `bg-ribbon-500`)
* **Accent (CTA/alerts)**: `flamingo.400/500`
* **Success/positive**: `aquamarine.500`
* **Info**: `ribbon.500`
* **Warning**: `sunflower.500`
* **Magenta/fuchsia highlight**: `magenta.500`
* **Neutral text**: `vulcan.700` for paragraphs, `vulcan.500` for secondary, `vulcan.300` for borders.

---

## 7) Helpful one-liners

* Headings on the fly (without the class alias):
  `className="text-[118px] leading-[1.04] tracking-[-0.02em] font-black"`
* Body text token on the fly:
  `className="text-[18px] leading-[1.4]"`

---

## 8) What to build next with these tokens

* Convert common components (Hero, SectionTitle, Card, Button) to use `hd-*` / `text-*b` and the semantic colors above.
* If you want fluid type later, swap the `fontSize` values for `clamp()` strings (same keys), keeping `letterSpacing` and `lineHeight` as-is.
